บทที่ 5: 2D SLAM & Localization

5.1 การสร้างแผนที่ (SLAM)

SLAM (Simultaneous Localization and Mapping) คือการให้หุ่นยนต์วิ่งไปในพื้นที่ที่ไม่รู้จัก สร้างแผนที่ และระบุตำแหน่งตัวเองไปพร้อมๆ กัน โดยเราจะใช้ slam_toolbox ซึ่งเป็นมาตรฐานของ ROS2

  1. เปิด Terminal (บน VNC หรือ SSH) เข้า Docker แล้วรัน:
    ros2 launch slam_mapping slam_toolbox.launch.py
  2. เปิดโปรแกรม RViz2 บนหน้าจอ VNC เพื่อดูแผนที่แบบ Real-time
  3. ใช้จอยคอนโทรลเลอร์ หรือรันคีย์บอร์ด teleop ขับหุ่นยนต์ไปรอบๆ ห้อง จนกว่าจะได้แผนที่ที่สมบูรณ์ (ผนังห้องครบถ้วน)

5.2 การบันทึกแผนที่

เมื่อเราสแกนห้องเสร็จแล้ว ต้องบันทึกข้อมูลไว้เพื่อใช้ในระบบนำทางต่อไป (จะได้ไฟล์ .yaml และ .pgm)

# รันคำสั่งเซฟใน Terminal ใหม่ (ที่อยู่ใน Docker)
ros2 run nav2_map_server map_saver_cli -f /home/pi/hiwonder_ws/src/mentorpi_maps/maps/my_room

5.3 การระบุตำแหน่ง (AMCL Localization)

หลังจากมีแผนที่แล้ว เมื่อหุ่นยนต์เปิดขึ้นมาใหม่ มันจะไม่รู้ว่าตัวเองอยู่ตรงไหนของแผนที่ เราต้องใช้ AMCL (Adaptive Monte Carlo Localization) เพื่อช่วยระบุตำแหน่ง

  • ในโปรแกรม RViz2 ให้ใช้เครื่องมือ 2D Pose Estimate (ลูกศรสีเขียว) คลิกแล้วลากบนแผนที่เพื่อบอกหุ่นยนต์ว่า "ตอนนี้คุณอยู่ตรงนี้นะ และหันหน้าไปทางนี้"
  • เมื่อหุ่นยนต์เริ่มขยับ Lidar จะจับคู่กับผนังจริงและแผนที่ ทำให้ตำแหน่ง (Particles) แม่นยำขึ้น
