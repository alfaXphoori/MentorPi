บทที่ 1: การเตรียมความพร้อม (Environment Setup)

1.0 ทำความรู้จักฮาร์ดแวร์ (Hardware Overview)

ก่อนเริ่มการเชื่อมต่อ เรามาทำความรู้จักส่วนประกอบสำคัญของหุ่นยนต์ MentorPi M1 กันก่อนครับ

  • Raspberry Pi 5: บอร์ดคอมพิวเตอร์หลักที่ทำหน้าที่เป็น "สมอง" ควบคุมทุกระบบ
  • Expansion Board: แผงวงจรส่วนขยายที่เชื่อมต่อกับมอเตอร์ แบตเตอรี่ และเซนเซอร์ระดับล่าง
  • Mecanum Wheels: ล้ออัจฉริยะ 4 ทิศทาง พร้อมมอเตอร์เกียร์โลหะและ Encoder
  • TOF Lidar (STL-19P): เซนเซอร์หมุนอยู่ด้านบน ทำหน้าที่สแกนพื้นที่รอบตัวแบบ 360 องศา (SLAM)
  • AI Monocular Camera: กล้องด้านหน้าสำหรับตรวจจับภาพและใช้ทำงานร่วมกับโมเดล AI (เช่น YOLO)

1.1 การเชื่อมต่อเครือข่าย (Network Connection)

หุ่นยนต์ MentorPi M1 จำเป็นต้องเชื่อมต่อเครือข่ายเพื่อให้เราสามารถส่งคำสั่งและดูข้อมูลเซนเซอร์ได้แบบไร้สาย โดยระบบรองรับ 2 โหมดหลัก ดังนี้:

โหมดที่ 1: AP Mode (Access Point) - แนะนำสำหรับการเริ่มต้น

ในโหมดนี้ หุ่นยนต์จะทำหน้าที่เป็น Router ปล่อยสัญญาณ Wi-Fi ออกมาเอง เหมาะสำหรับใช้ในกรณีที่ห้องเรียนไม่มีอินเทอร์เน็ต หรือต้องการเชื่อมต่อแบบ 1 ต่อ 1

  1. เปิดสวิตช์หุ่นยนต์ และรอประมาณ 1-2 นาทีจนกว่าระบบปฏิบัติการจะบูตเสร็จ
  2. ใช้คอมพิวเตอร์ของคุณค้นหาสัญญาณ Wi-Fi ที่ชื่อขึ้นต้นด้วย HW-xxxxxx (เช่น HW-123456)
  3. กรอกรหัสผ่านเริ่มต้น: hiwonder
  4. เมื่อเชื่อมต่อสำเร็จ IP Address ของหุ่นยนต์จะถูกกำหนดเป็น 192.168.149.1 เสมอ

โหมดที่ 2: LAN Mode (Local Area Network)

ให้หุ่นยนต์ไปเกาะ Wi-Fi ของห้องเรียน เหมาะสำหรับการใช้อินเทอร์เน็ต (เช่น โหลดโมเดล AI) หรือเมื่อมีหุ่นยนต์หลายตัวในเครือข่ายเดียวกัน

  1. เชื่อมต่อหุ่นยนต์ผ่าน AP Mode ก่อน หรือเสียบสาย LAN / จอ HDMI
  2. ใช้คำสั่งเชื่อมต่อ Wi-Fi ผ่าน Terminal หรือตั้งค่าผ่านหน้า VNC Desktop
  3. เมื่อหุ่นยนต์ได้ IP Address ใหม่จาก Router ของห้องเรียน (เช่น 192.168.1.xxx) ให้จด IP นั้นไว้ใช้สำหรับการรีโมทในครั้งต่อไป

💡 Tip วิทยากร: แนะนำให้ใช้เครื่องมืออย่าง Advanced IP Scanner หรือดูผ่านแอปมือถือ Fing เพื่อหา IP ของหุ่นยนต์ในกรณีที่ใช้โหมด LAN ครับ

1.2 การเข้าถึงหุ่นยนต์ (Remote Access)

เมื่อเชื่อมต่อ Wi-Fi สำเร็จ เราต้องเข้าไปสั่งงานบอร์ด Raspberry Pi 5 ผ่านเครือข่าย โดยไม่ต้องต่อจอภาพหรือคีย์บอร์ดเข้ากับตัวหุ่นยนต์โดยตรง (Headless Setup) ซึ่งเราจะใช้เครื่องมือ 2 รูปแบบ:

1. SSH (Secure Shell) - สำหรับการสั่งงานผ่าน Command Line

SSH คือการรีโมทเข้าไปพิมพ์คำสั่ง (Terminal) เป็นวิธีที่กินทรัพยากรน้อยและรวดเร็วที่สุด เหมาะสำหรับการสั่งรันโค้ด หรือตรวจสอบสถานะระบบ

  1. ผู้ใช้ Windows: สามารถเปิดโปรแกรม Command Prompt (cmd) หรือ PowerShell
  2. ผู้ใช้ Mac / Linux: สามารถเปิดโปรแกรม Terminal ได้เลย
  3. พิมพ์คำสั่งต่อไปนี้ (สมมติว่าใช้ AP Mode IP: 192.168.149.1):
ssh pi@192.168.149.1
  • Username: pi
  • Password: pi (ขณะพิมพ์รหัสผ่าน หน้าจอจะไม่แสดงตัวอักษร ให้พิมพ์แล้วกด Enter ได้เลย)

2. VNC Viewer (Virtual Network Computing) - สำหรับการดูหน้าจอ Desktop

VNC อนุญาตให้เราเห็นหน้าจอ Ubuntu ของหุ่นยนต์แบบกราฟิก (GUI) เหมือนเราไปนั่งอยู่หน้าจอจริงๆ ซึ่งจำเป็นมากเมื่อเราต้องทำ Computer Vision (ดูภาพจากกล้อง) หรือดูแผนที่ (SLAM)

  1. ดาวน์โหลดและติดตั้งโปรแกรม RealVNC Viewer ลงบนคอมพิวเตอร์ของคุณ
  2. เปิดโปรแกรม VNC Viewer ขึ้นมา
  3. ในช่อง Address Bar (Enter a VNC Server address or search) ให้พิมพ์ IP ของหุ่นยนต์: 192.168.149.1 แล้วกด Enter
  4. เมื่อมีหน้าต่างเตือนเรื่องความปลอดภัย ให้กด Continue
  5. ระบบจะถามรหัสผ่าน (Username จะเป็น pi อัตโนมัติ) ให้กรอกรหัสผ่าน: pi

📝 Note: เมื่อเข้าสู่หน้าจอ VNC สำเร็จ คุณจะเห็นหน้าต่าง Desktop ของ Ubuntu 22.04 คุณสามารถเปิด Terminal ในนี้เพื่อทำงานต่างๆ ได้เช่นกัน

1.3 สภาพแวดล้อม Docker (Docker Environment)

สำคัญมาก: โค้ดและไลบรารี ROS2 ทั้งหมดของ MentorPi M1 ไม่ได้ถูกติดตั้งบนเครื่อง Raspberry Pi โดยตรง แต่ถูกติดตั้งไว้ใน Docker Container (สภาพแวดล้อมจำลอง) เพื่อป้องกันไม่ให้ระบบหลักพังเมื่อมีการแก้ไขไฟล์ และช่วยให้ระบบทำงานได้เสถียรที่สุด

การเข้าสู่ Docker Container

ทุกครั้งที่คุณเปิด Terminal ใหม่ (ไม่ว่าจะผ่าน SSH หรือใน VNC) คุณต้องเข้าสู่ Docker ก่อนเสมอ จึงจะสามารถใช้คำสั่ง ros2 ได้

  1. เปิด Terminal
  2. ไปที่โฟลเดอร์ของโปรเจกต์ (ปกติเมื่อล็อกอินเข้ามาจะอยู่ที่โฟลเดอร์ Home /home/pi อยู่แล้ว):
    cd ~
  3. รันสคริปต์เพื่อเข้าสู่ Docker:
    ./enter_docker.sh

วิธีสังเกตว่าอยู่ใน Docker หรือยัง?

ให้สังเกตที่บรรทัดคำสั่ง (Prompt) ใน Terminal:

  • อยู่นอก Docker (Host OS): pi@MentorPi:~$
  • อยู่ใน Docker แล้ว: root@MentorPi:~/hiwonder_ws# (สังเกตว่าชื่อ User จะเปลี่ยนเป็น root และ Path จะเปลี่ยนเป็น hiwonder_ws)

Workspace (พื้นที่ทำงาน)

โค้ดทั้งหมดที่ใช้ควบคุมหุ่นยนต์จะถูกเก็บไว้ที่ /home/pi/hiwonder_ws/ ภายใน Docker ซึ่งจะเชื่อมโยง (Mount) กับไฟล์ในระบบหลัก หากคุณแก้ไขไฟล์ในโฟลเดอร์นี้ผ่าน VNC ไฟล์ใน Docker ก็จะเปลี่ยนตามไปด้วย

⚠️ ข้อควรระวัง: หากคุณพิมพ์คำสั่ง ros2 ... แล้วเจอ Error ว่า "command not found" นั่นแปลว่าคุณลืมรันคำสั่ง ./enter_docker.sh ครับ!
