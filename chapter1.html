บทที่ 1: การเตรียมความพร้อม (Environment Setup)

1.0 ทำความรู้จักฮาร์ดแวร์ (Hardware Overview)

ก่อนเริ่มการเชื่อมต่อ เรามาทำความรู้จักส่วนประกอบสำคัญของหุ่นยนต์ MentorPi M1 กันก่อนครับ

  • Raspberry Pi 5: บอร์ดคอมพิวเตอร์หลักที่ทำหน้าที่เป็น "สมอง" ควบคุมทุกระบบ
  • Expansion Board: แผงวงจรส่วนขยายที่เชื่อมต่อกับมอเตอร์ แบตเตอรี่ และเซนเซอร์ระดับล่าง
  • Mecanum Wheels: ล้ออัจฉริยะ 4 ทิศทาง พร้อมมอเตอร์เกียร์โลหะและ Encoder
  • TOF Lidar (STL-19P): เซนเซอร์หมุนอยู่ด้านบน ทำหน้าที่สแกนพื้นที่รอบตัวแบบ 360 องศา (SLAM)
  • AI Monocular Camera: กล้องด้านหน้าสำหรับตรวจจับภาพและใช้ทำงานร่วมกับโมเดล AI (เช่น YOLO)

1.1 การเชื่อมต่อเครือข่าย (Network Connection)

หุ่นยนต์ MentorPi M1 รองรับการเชื่อมต่อผ่าน Wi-Fi เพื่อให้เราสามารถส่งคำสั่งและดูข้อมูลเซนเซอร์ได้แบบไร้สาย โดยมี 2 โหมดหลัก:

  • AP Mode (Access Point): หุ่นยนต์ปล่อยสัญญาณ Wi-Fi ออกมาเอง (ชื่อ HW-xxxx) เหมาะสำหรับใช้เมื่อไม่มีอินเทอร์เน็ตในห้องเรียน
  • LAN Mode: เชื่อมต่อหุ่นยนต์เข้ากับ Router ของห้องเรียน เหมาะสำหรับการใช้งานพร้อมกันหลายตัว

1.2 การเข้าถึงหุ่นยนต์ (Remote Access)

เมื่อเชื่อมต่อ Wi-Fi สำเร็จ เราต้องเข้าไปสั่งงานบอร์ด Raspberry Pi 5 ที่อยู่บนตัวหุ่นยนต์ ผ่านเครื่องมือ 2 ตัวหลัก:

1. SSH (Secure Shell)

ใช้สำหรับพิมพ์คำสั่งผ่าน Terminal (Command Line) โดยตรง เหมาะสำหรับการเปิดใช้งาน Node ต่างๆ ของ ROS2

ssh pi@192.168.149.1
# รหัสผ่านเริ่มต้นคือ: pi

2. VNC Viewer (Remote Desktop)

ใช้สำหรับดูหน้าจอ Desktop ของหุ่นยนต์แบบกราฟิก (GUI) จำเป็นอย่างยิ่งเมื่อเราต้องการดูภาพจากกล้อง หรือดูการสร้างแผนที่ (Map)

1.3 สภาพแวดล้อม Docker (Docker Environment)

Hiwonder ออกแบบให้โปรแกรมของหุ่นยนต์ทั้งหมดทำงานอยู่ภายใน Docker Container เพื่อความเสถียรและป้องกันระบบปฏิบัติการหลัก (Host OS) เสียหาย

คำสั่งพื้นฐานที่ต้องรู้

  • เข้าสู่ Docker Container ของหุ่นยนต์:
    ./enter_docker.sh
  • ทุกครั้งที่จะรันคำสั่ง ROS2 จะต้องอยู่ในโหมด Docker เสมอ (สังเกตได้จากชื่อ User ใน Terminal จะเปลี่ยนไป)
